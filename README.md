# 🔍 RUC-Detect: Modular Hallucination Detection

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)
[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/)

Official project code for detecting Large Language Model (LLM) hallucinations through rigorous component-based verification. This repository provides an end-to-end framework integrating RAG-based factual retrieval, Natural Language Inference (NLI) scoring, and SelfCheckGPT sampling heuristics.

## 🏗 Project Architecture

Our detection pipeline evaluates hallucinations by fusing three distinct analytical methods logically weighted to produce a final confidence scope threshold:

1. **RAG Retrieval Verification (40% Weighting)**: Computes embedded cosine similarity checks against `FAISS` structured encyclopedic documents chunks generated from reliable public data to score explicit deviation factors.
2. **NLI Entailment Processing (30% Weighting)**: Uses cross-attention logic via `DeBERTa-v3` (`utils/nli.py`) inspired by the **FactCC** approach. Maps localized premise vs. hypothesis entailment limits directly identifying contradictions.
3. **SelfCheckGPT-NLI (30% Weighting)**: Employs stochastic sampling stability techniques (`utils/selfcheck.py`). Tests if distinct samples generated by origin models logically drift, indicating synthetic generation hallucinations.

All metrics are consolidated into `utils/taxonomy.py` which dynamically attributes structural tags: `extrinsic`, `intrinsic`, and `semantic_drift`.

## 🚀 Quickstart & Deployment

### 1. Environment Setup

We provide a zero-conflict Conda environment:
```bash
conda env create -f environment.yml
conda activate ruc-detect
```
Alternatively, using `requirements.txt`:
```bash
pip install -r requirements.txt
```

*(Critical)* **Download spaCy core dependencies:**
```bash
python -m spacy download en_core_web_sm
```

### 2. Build the FAISS Document Index

The pipeline utilizes structured retrieved documents natively. Build the local offline vector cache using HuggingFace Wikipedia splits:

```bash
python build_index.py --sample 5000
```
> **Note:** Building the full sample (`--sample 50000`) compiles over 200,000 overlapping chunks, spanning roughly ~2 hours locally. Specifying `--sample 5000` is highly recommended for testing efficiency.

### 3. Run the Gradio UI Demo
```bash
python app.py
```
Open `http://localhost:7860` in your browser. Note: Gradio is structured to query the FastAPI service through `SPACE_API_URL` environment variables if deployed dynamically on HuggingFace Spaces.

### 4. Deploy the API Server via Docker
To run the production-ready FastAPI service featuring robust structured-logging endpoints and caching arrays natively via multi-layered dockerization:
```bash
docker-compose up --build -d
```
Access the immersive OpenAPI Endpoint definitions natively at: `http://localhost:8000/docs`.

*(For detailed CI/CD schemas and deployments, consult our `README_DEPLOY.md` file!)*

## 🔬 Reproducing Benchmark Results

To evaluate and reproduce analytics on the **HaluEval** QA benchmark natively, utilize the integrated script to rapidly execute the pipeline on randomized test subsets generating stratified analytic exports.

```bash
# Execute evaluation on 200 items generating automated threshold sweeps
python evaluate.py

# Or perform an accelerated validation diagnostic (50 samples format) 
python evaluate.py --quick
```

Outputs are automatically pushed identically into the `/results` repository formatted comprehensively as structured `.json` metric-payloads.
